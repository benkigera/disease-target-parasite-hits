Disease → Target → Pathogen-Hit Map

Given a list of diseases our goal is to find their top 100 associated targets from the Open Targets API, map each target to a UniProt Swiss-Prot ID, then check whether those host proteins are interacting with any pathogens in HPIDB.

input disease ID list
step 1–4 disease → target map
step 5–6 target → UniProt Swiss-Prot ID
step 7–11 disease → target → pathogen-hit map
Open Targets scores and ranks targets through their genetic association data. UniProt is the bridge that lets Open Targets and HPIDB talk to each other.
01

Find the documentation

Our first step is to create a disease → target map. Before writing any code, let's find out what Open Targets actually offers for querying disease-target associations.

  • Open Targets GraphQL API docs
  • GraphQL playground
→ Read the docs first. Know what fields are available before building the query.
02

Head to the GraphQL playground and construct the query

Let's head to the playground so we can construct our query and get exactly what we need. We'll use the disease query — it accepts a disease ID and can return several things. In our case we're interested in: disease name, disease ID, UniProt IDs for each returned target, target name, and target ID.

We can try it with one disease first, see what the data looks like and how it's returned — the shape of the response. We can also add the disease ID as a variable and name our query something like fetch_top_targets. 

query fetch_top_targets($disease_id: String!) {
  disease(efoId: $disease_id) {
    id
    name
    associatedTargets(page: { index: 0, size: 100 }) {
      rows {
        score
        target {
          id
          approvedSymbol
          approvedName
          proteinIds {
            id
            source
          }
        }
      }
    }
  }
}

Variables panel:

{ "disease_id": "EFO_0000398" }
→ associatedTargets.rows is a list. Each row has a score and a nested target object with a proteinIds list inside it.
03

Go back to the docs — fetch for a single disease in Python

Now let's go back to the documentation and see how we fetch targets using Python and print the result for a single disease. See if it works. 

import requests

OPEN_TARGETS_URL = "https://api.platform.opentargets.org/api/v4/graphql"

fetch_top_targets_query = """
query fetch_top_targets($disease_id: String!) {
  disease(efoId: $disease_id) {
    id
    name
    associatedTargets(page: { index: 0, size: 100 }) {
      rows {
        score
        target {
          id
          approvedSymbol
          approvedName
          proteinIds { id source }
        }
      }
    }
  }
}
"""

response = requests.post(
    OPEN_TARGETS_URL,
    json={
        "query": fetch_top_targets_query,
        "variables": {"disease_id": "EFO_0000398"},
    },
)

data = response.json()
print(data)
→ The Python request mirrors exactly what worked in the playground — same query, same variable, just sent as JSON over HTTP.

Iterate over our disease list and store as JSON

From there we can iterate over our full list of disease IDs, fetch each one, and store the results as a JSON file. We want something like this shape, which we can store as-is for now:

Note: Open Targets doesn't return a rank field directly. We create it from the row position using enumerate — the list comes back pre-sorted by score, so position = rank. 

for rank, row in enumerate(rows, start=1):
    target = row["target"]

    swissprot_ids = [
        p["id"] for p in target["proteinIds"]
        if p["source"] == "uniprot_swissprot"
    ]

    top_targets.append({
        "target_name":        target["approvedName"],
        "target_id":          target["id"],
        "rank":               rank,
        "association_score":  row["score"],
        "uniprot_swissprot_id": swissprot_ids[0] if swissprot_ids else None,
    })
→ Once this JSON exists, all later scripts load it instead of calling Open Targets again.

06

Pull all Swiss-Prot IDs from the map

Once we have that, let's collect all the Swiss-Prot UniProt IDs from our disease-target dataset into a set. This is what we'll use to look up matches in HPIDB. 

host_target_uniprot_ids = {
    target["uniprot_swissprot_id"]
    for disease in diseases
    for target in disease["top_targets"]
    if target.get("uniprot_swissprot_id")
}
→ A set gives us O(1) lookup later — no looping through the full target list for every HPIDB row.
07

Load HPIDB — read_text, split lines, delegate per-row logic

We'll then check HPIDB to see if our host targets are interacting with any pathogens. Instead of opening the file with a context manager, we use read_text().splitlines() to get all lines at once, then split each line on \t to get the columns. We delegate the per-row hit-finding logic to its own function to keep things clean. 

def collect_pathogen_hits_by_target(target_uniprot_ids):
    hits = {}
    for line in (
        Path("data/input/hpidb2.mitab/hpidb2.mitab_plus.txt")
        .read_text(encoding="utf-8", errors="replace")
        .splitlines()
    ):
        columns = line.split("\t")

        result = get_pathogen_hit_for_target(columns, target_uniprot_ids)
        if result:
            host_id, pathogen_hit = result
            hits.setdefault(host_id, []).append(pathogen_hit)

    return hits

08

Parse each row — figure out which side is human

Each interaction row has two proteins. MITAB accessions look like uniprotkb:P23458 and taxids look like taxid:9606(Homo sapiens) — so we need to parse out just the ID portion from each. 

def fetch_hits_for_a_single_target(columns, target_uniprot_ids):
    # accessions are "uniprotkb:P23458" — split on ":" and take the last part
    protein_a = columns[15].split(":")[-1]
    protein_b = columns[16].split(":")[-1]

    # taxids are "taxid:9606(Homo sapiens)" — split on ":" then on "("
    taxid_a = columns[9].split(":")[1].split("(")[0] if ":" in columns[9] else None
    taxid_b = columns[10].split(":")[1].split("(")[0] if ":" in columns[10] else None

    # columns 17 & 18 — organism kingdom for each side (ANIMAL, BACTERIA, VIRUS…)
    kingdom_a = columns[17]
    kingdom_b = columns[18]

    # protein A is human, protein B is the pathogen
    if taxid_a == "9606" and taxid_b != "9606":
        host_id          = protein_a
        pathogen_id      = protein_b
        pathogen_taxid   = taxid_b
        pathogen_name    = columns[25]
        pathogen_kingdom = kingdom_b

    # protein B is human, protein A is the pathogen
    elif taxid_b == "9606" and taxid_a != "9606":
        host_id          = protein_b
        pathogen_id      = protein_a
        pathogen_taxid   = taxid_a
        pathogen_name    = columns[24]
        pathogen_kingdom = kingdom_a

    else:
        return ""  # human-human or pathogen-pathogen — skip

    # keep only viral pathogen hits
    if pathogen_kingdom != "VIRUS":
        return ""

    if host_id not in target_uniprot_ids:
        return ""  # not one of our targets — skip

    pathogen_hit = {
        "pathogen_molecule": pathogen_name,
        "pathogen_id": pathogen_id,
        "pathogen_taxid": pathogen_taxid,
        "pathogen_kingdom": pathogen_kingdom,
    }
    return host_id, pathogen_hit
→ Whichever side is the pathogen, we grab its kingdom from column 17 or 18 and bail out unless it's VIRUS — so only viral interactions make it into the map.
09

Attach hits back and save the final map

We wire it together in main() — call collect to get hits_by_target, walk the disease-target map, look up each target's hits by its UniProt ID, attach the list, re-rank with the viral signal, then write the result to disk. 

def main():
    hits_by_target = collect_pathogen_hits_by_targets(target_uniprot_ids)

    for disease in disease_list:
        for target in disease["top_targets"]:
            target_id     = target.get("uniprot_swissprot")
            pathogen_hits = hits_by_target.get(target_id, [])
            target["pathogen_hits"] = pathogen_hits

    add_viral_informed_target_rank(disease_list)

    Path("data/outputs/disease_target_pathogen_map.json").write_text(
        json.dumps(disease_list, indent=2)
    )

main()
→ Attaching the hits answers which targets have viral evidence — the next step turns that evidence into a re-ranking.
10

Re-rank targets with the viral signal

Open Targets ranks purely on genetic association. But now that we know which targets actually interact with viruses, we can nudge that ranking — a target with strong viral evidence should climb. That's what add_viral_informed_target_rank() does, per disease.

For each target we count its viral molecule hits and the number of distinct viral source taxa behind them, then carry the original association score forward as association_score: 

for target in targets:
    pathogen_hits = target.get("pathogen_hits", [])

    viral_source_taxa = {
        hit["pathogen_taxid"]
        for hit in pathogen_hits
        if hit.get("pathogen_taxid")
    }

    target["association_score"]           = target["score"]
    target["viral_molecule_hit_count"]    = len(pathogen_hits)
    target["viral_source_taxa_count"]     = len(viral_source_taxa)
    target["supporting_interaction_count"] = len(pathogen_hits)

Then we normalise each target's hit count against the busiest target in that disease, and add up to a 0.2 bonus on top of the association score — that's the viral_informed_score: 

max_viral_hits = max(
    (t["viral_molecule_hit_count"] for t in targets),
    default=0,
)

for target in targets:
    viral_hit_bonus = (
        target["viral_molecule_hit_count"] / max_viral_hits
        if max_viral_hits else 0
    )
    target["viral_informed_score"] = (
        target["association_score"] + 0.2 * viral_hit_bonus
    )

Finally we sort by that new score to assign a viral_informed_rank, and record how many places each target moved versus its original Open Targets position as rank_gain: 

ranked_targets = sorted(
    targets,
    key=lambda target: target["viral_informed_score"],
    reverse=True,
)

for viral_rank, target in enumerate(ranked_targets, start=1):
    target["viral_informed_rank"] = viral_rank
    target["rank_gain"] = target["open_target_rank"] - viral_rank
→ The bonus caps at 0.2, so viral evidence can reshuffle the list without ever drowning out the genetic association. rank_gain makes the effect legible — a positive number means the virus data pushed that target up.

Debugging rules

  • If the value is a list, loop before using string keys
  • Use json.dumps(v, indent=2) for readable output
  • Do not print None for every missed HPIDB row
  • Use reader.py to inspect one MITAB row by column index
